# HexForge 未完成功能开发清单

## 一、编辑功能

### 1.0 键盘输入 - 十六进制区域
**需求描述：**
- 在十六进制显示区域直接键入数据修改文件
- 支持输入 0-9, A-F, a-f 字符
- 每个字节由两个十六进制字符组成（高4位、低4位）
- 输入第一个字符后光标保持在当前字节，输入第二个字符后自动移动到下一字节
- 输入超出文件末尾时自动扩展文件大小（仅插入模式）

**目标：**
- 用户可以通过键盘直接编辑十六进制数据
- 输入体验流畅，符合主流十六进制编辑器习惯

**测试方法：**
1. 打开测试文件，点击十六进制区域
2. 键入 "A"，验证高4位变为 A，光标仍在当前字节
3. 键入 "B"，验证完整字节变为 AB，光标移动到下一字节
4. 键入 "0-9"，验证正常输入
5. 键入 "G/H/I" 等非法字符，验证无响应
6. 按退格键，验证删除字符行为

**技术实现：**
- 在 `HexView` 中重写 `keyPressEvent`
- 创建 `InputHandler` 类处理输入验证
- 扩展 `CursorModel` 支持 nibble（半字节）级别定位

---

### 1.0.1 键盘输入 - ASCII 区域
**需求描述：**
- 在 ASCII 显示区域直接键入数据修改文件
- 支持输入任意 ASCII 字符（32-126）
- 每个字符直接映射为对应字节值
- 输入后光标自动移动到下一字节
- 不可打印字符（0-31, 127-255）在 ASCII 区域显示为 "."，但仍可编辑

**目标：**
- 用户可以通过 ASCII 区域快速输入文本数据
- 支持在 ASCII 区域编辑非可打印字符对应的字节

**测试方法：**
1. 打开测试文件，点击 ASCII 区域
2. 键入 "Hello"，验证对应的十六进制值变为 48 65 6C 6C 6F
3. 键入中文字符，验证无响应或提示（仅支持 ASCII）
4. 按退格键，验证删除字符行为
5. 在十六进制区域修改为非可打印值，验证 ASCII 区域显示 "."
6. 点击 "." 位置键入字符，验证可以修改

**技术实现：**
- 在 `HexView` 中根据当前列（0=Hex, 1=ASCII）选择输入处理方式
- ASCII 输入直接转换为字节值

### 1.0.2 编辑模式切换（覆盖/插入）
**需求描述：**
- 支持**覆盖模式（Overwrite）**：默认模式，键入数据覆盖当前位置
- 支持**插入模式（Insert）**：键入数据在光标位置插入，后续数据后移
- 按 `Insert` 键在两种模式间切换
- 状态栏显示当前模式：`OVR` 或 `INS`
- 切换模式时光标样式变化（覆盖=块状，插入=竖线）

**目标：**
- 用户可以根据需要选择编辑模式
- 模式切换直观、有反馈

**测试方法：**
1. 打开测试文件，验证默认为覆盖模式，状态栏显示 `OVR`
2. 按 `Insert` 键，验证切换为插入模式，状态栏显示 `INS`
3. 在覆盖模式下修改数据，验证原数据被覆盖，文件大小不变
4. 在插入模式下修改数据，验证数据被插入，文件大小增加
5. 连续按 `Insert` 键，验证模式来回切换
6. 保存文件后重新打开，验证模式重置为覆盖模式

**技术实现：**
- 在 `HexView` 中添加 `_edit_mode` 状态变量
- 添加 `toggleEditMode()` 方法
- 修改 `FileHandle.write()` 和 `FileHandle.insert()` 的调用逻辑

---

### 1.0.3 复制功能
**需求描述：**
- 支持复制选中的数据到系统剪贴板
- 复制格式根据选中区域和显示模式智能决定：
  | 选中区域 | 显示模式 | 复制格式 |
  |---------|---------|---------|
  | 十六进制区域 | HEX | `"00 01 02 FF"` (空格分隔) |
  | 十六进制区域 | BINARY | `"00000000 00000001 ..."` (8位二进制) |
  | 十六进制区域 | OCTAL | `"000 001 002 ..."` (3位八进制) |
  | ASCII 区域 | 任意 | 原始 ASCII 文本 |
- 快捷键：`Ctrl+C`
- 右键菜单：复制

**目标：**
- 用户可以方便地复制数据到其他应用程序
- 复制格式符合预期，便于粘贴使用

**测试方法：**
1. 选中十六进制区域的几个字节，按 `Ctrl+C`
2. 粘贴到文本编辑器，验证格式为 `"00 01 02 FF"`
3. 切换到二进制显示模式，复制，验证格式为二进制
4. 选中 ASCII 区域的字符，复制，验证为纯文本
5. 选中跨行数据，复制，验证格式正确
6. 无选中时复制，验证无操作或提示

**技术实现：**
- 创建 `ClipboardManager` 类处理剪贴板操作
- 在 `main_window.py` 中添加复制菜单和快捷键
- 根据当前选择区域和显示模式选择复制格式

### 1.0.5 粘贴功能
**需求描述：**
- 支持从系统剪贴板粘贴数据
- 自动识别剪贴板内容格式：
  1. 十六进制字符串（`"00 01 FF"` 或 `"00FF"` 或 `"00, 01, FF"`）
  2. 纯 ASCII 文本
  3. 二进制数据（MIME type `application/octet-stream`）
- 粘贴行为根据编辑模式：
  - 覆盖模式：从光标位置开始覆盖数据
  - 插入模式：在光标位置插入数据
- 快捷键：`Ctrl+V`
- 右键菜单：粘贴
- 粘贴前检查数据长度，超出文件末尾时提示或自动扩展

**目标：**
- 用户可以方便地从其他来源粘贴数据
- 自动识别格式减少用户操作

**测试方法：**
1. 复制十六进制字符串 `"00 01 02 FF"` 到剪贴板
2. 在编辑器中粘贴，验证正确解析为 4 个字节
3. 复制纯文本 `"Hello"` 到剪贴板，粘贴，验证正确转换
4. 在插入模式粘贴，验证数据被插入
5. 在覆盖模式粘贴，验证数据被覆盖
6. 粘贴非法格式（如 `"GG HH"`），验证提示错误
7. 粘贴数据超出文件末尾，验证自动扩展

**技术实现：**
- 在 `ClipboardManager` 中实现格式识别逻辑
- 添加 `parseHexString()`, `parseBinaryData()` 方法
- 根据编辑模式调用 `FileHandle.write()` 或 `FileHandle.insert()`
